Custom Proprietary CRM

Purpose

This application replaces a legacy desktop CRM with a custom web based responsive CRM. Additional functionality for geospatial representation of customers, and advanced analytics is required.

Use Case

Mobile sales force needs an application that empowers strategic selling and tracking of sales activities while on the road. Multiple roles are used including Sales User, Sales Manager, Sales Admin and Developer.

Technology

Database- application data is stored in a SQL database and all CRUD operations are handled through a REST API, Knowledge Base data is stored in a MongoDB and accessed through a REST API

Web Server- Node.js with Express.js MVC architecture

Testing- Qunit

Authentication- Auth0 connector with LDAP AD

Client- Single page responsive web application using JQuery, Knockout, and Bootstrap

Data Visualization- Leaflet, D3

Target Devices- Phone, Tablet, Desktop

My Role

Serve as product owner for the associated REST API project

Scrum Master

Design of UI an UX for the entire application

Implement Auth0 login for authentication integrated with LDAP and AD, with login required for for all pages

Create a responsive viewport layout with tab navigation including a dashboard, customer search, shipment search, customer and contact profiles, and user administration of role based visibility.

Dashboard- home page for users when they login and organized into a three column responsive viewport that displays an accordion panel with grouped and prioritized activities, an accordion panel containing a grouped display of the next three weeks of sales activities, and a panel displaying conversation ratios as well as an accordion view of the sales pipeline.

Customer Search- a three panel responsive viewport containing a filter panel, grid panel and map panel. Users to search for customer using multiple combinations of search criteria. The map is interactive and users can open customer profiles from the grid or the map, as well as searching using input fields or map zoom and click functionality.

Shipment Search- a three panel responsive viewport with a filter panel, grid panel, and detailed shipment panel. The user can search using multiple combinations of search criteria via a input fields, and can open the any of the customer profiles directly from the shipment, as well as entering sales activities without having to navigate to the customer profile.

Customer Profile- a complete view of the customer with role based CRUD operations for all customer information.

Contact Profile- a complete view the contact with CRUD operations for all contact information.

User Administration- a responsive grid for role based administration of available information and CRUD functions.

Knowledge Base- write user documentation available from within the application with fuzzy search.

End user training.

Real Time Pricing Application

Purpose

The purpose of this application is to provide call center operations users with empirical data and real time market trends for spot rate pricing.

Use Case

Users access the application through a link provided in a Windows Forms desktop application which will provide basic information via search parameters in the URL. The application then calls a REST API to gather all information available and provide realtime geospatial representation of all lanes including waypoints of the current shipment accompanied by empirical shipment data. Both empirical data and current market trends including fuel surcharges and market scarcity are then used to calculate guidelines for spot rate pricing. The user will then select multiple options for moving a shipment and generate multiple price and time quotes that are saved back into the TMS system.

Technology

Version Control- GIT

Database- application data is stored in a SQL database and all CRUD operations are handled through a REST API, Knowledge Base data is stored in a MongoDB and accessed through a REST API

Web Server- Node.js

Testing- Qunit

Authentication- Auth0 connector with LDAP AD

Client- Single page responsive web application using JQuery, Aurelia, and Bootstrap

Data Visualization- Leaflet

Target Devices- Desktop, 70” smart screen

My Role

Serve as product owner for the associated REST API project

Scrum Master

Design of UI an UX for the entire application

Implement Auth0 login for authentication integrated with LDAP and AD

Create a responsive viewport layout with four panels for display of shipment origin and destination information, map to display the current shipment with waypoints, detail grid of empirical data, and an interactive route selection and price generation panel.

Shipment Origin and Destination- information retrieved using AJAX via rest api, with user select input for airport in the area.

Map- a large Leaflet map displaying markers for origin and destination addresses, pins for selected airports, and routing for all ground and air lanes using waypoints.

Detail Grid- a sortable grid using DataTable displaying all historical shipments matching search criteria with expandable rows to display routing details.

Route Selection and Price Generation- a panel with multiple routing options that are selected by the user by clicking a check box. Each routing combination produces a total price, and individual lane price that can be saved by the user to create multiple quoting options for different routings.

Team Building

I am an active athlete in both team and individual sports and carry the same spirit of competition and desire to win for my teams in the workplace.

My experience has taught me that teams who are in a constant pursuit of excellence are acutely aware of their individual strengths and ability for continuous improvement. This understanding is then applied to both the tactical application of skills, as well as the teaching and sharing of knowledge enabling the growth of the team.

This approach requires a great deal of self-reflection and focus thus, enforcing a strong sense of peer accountability yielding highly motivated and self-managed teams. By establishing defined roles and implementing continuous feedback both from coaches or managers, as well as peers, individuals are more likely to become highly valued members of the team. When all members of a team are performing at their best the group synergy becomes visible and the ultimate output can exceed even the highest expectations.

Scrum Master

The focus of Scrum is to deliver small chunks of production-ready software and incrementally deliver ROI during the course of a project. This requires a commitment to defining a minimum viable product, which also has the added benefit of preventing over engineered software before receiving end-user feedback. This user feedback is not only valuable in grooming the product backlog, but is also an incredibly valuable change management and user adoption tool.

Sprint demos and daily standups are not unique to the Scrum process and are concepts that I have used in many projects. I have found great value in the sprint retrospective given my firm belief in continuous feedback and evaluation process. Sitting down as a team following every sprint demo allows for candid conversation about what we are doing well and what we need to improve upon; greatly increasing the growth and productivity of the team.
